# Business Model Canvas – **llm‑cache**

| **Building Block** | **Details** |
|--------------------|-------------|
| **Value Proposition** | • **Ultra‑low latency** inference for LLMs via intelligent request deduplication and result reuse.<br>• **Cost reduction** – up to 40 % lower compute spend by avoiding redundant token generation.<br>• **Plug‑and‑play** – drop‑in middleware for any vLLM‑compatible server (vLLM, SGLang, etc.) with minimal code changes.<br>• **Scalable persistence** – in‑memory + optional Redis/ClickHouse backend for multi‑node cache sharing.<br>• **Safety‑aware** – respects token‑level privacy flags; cache entries can be scoped per‑user or per‑tenant. |
| **Customer Segments** | 1. **LLM SaaS providers** (e.g., chat‑bot platforms, code‑completion APIs) seeking to improve SLA and reduce GPU bills.<br>2. **Enterprises** running private LLM inference clusters for internal tools (RAG, summarisation, support bots).<br>3. **Edge AI vendors** deploying quantised LLMs on devices where compute is scarce.<br>4. **ML Ops platforms** (e.g., HuggingFace Inference Endpoints, AWS SageMaker) that need a reusable caching component for their customers. |
| **Channels** | • **Open‑source repo** (GitHub `arkashira/llm-cache`) with detailed quick‑start docs.<br>• **Package distribution** – Docker image, Helm chart, and Python wheel on PyPI.<br>• **Developer evangelism** – blog posts, webinars, and sample integrations with vLLM & SGLang.<br>• **Partner integrations** – bundled with Axentx’s inference stack (vLLM, SGLang) and listed in their marketplace.<br>• **Enterprise sales** – direct outreach, PoC support, and custom SLA contracts. |
| **Revenue Streams** | 1. **Subscription SaaS tier** – hosted, fully‑managed llm‑cache service (per‑request or per‑GPU‑hour pricing).<br>2. **Enterprise license** – on‑premise binary with support & SLAs (annual fee + optional consumption‑based add‑on).<br>3. **Professional services** – integration consulting, performance tuning, and custom feature development.<br>4. **Marketplace royalties** – revenue share from third‑party platforms that embed llm‑cache as a plug‑in. |
| **Cost Structure** | • **Infrastructure** – cloud compute & storage for managed service (GPU nodes, Redis/ClickHouse clusters).<br>• **Engineering** – salaries for core dev, QA, and SRE teams; open‑source maintenance.<br>• **Licensing** – fees for any third‑party components (e.g., Redis Enterprise, monitoring tools).<br>• **Sales & Marketing** – developer outreach, content creation, and enterprise sales headcount.<br>• **Compliance & Security** – audits, privacy certifications, and data‑handling tooling. |
| **Key Resources** | • **Core codebase** – high‑performance caching engine written in Rust/Python, tightly integrated with vLLM & SGLang.<br>• **Dataset & telemetry** – Axentx’s 17 M+ LLM request logs used to train cache‑hit prediction models.<br>• **Talent** – senior systems engineers, ML‑ops specialists, and product managers.<br>• **Platform** – Axentx’s BRAIN (pgvector) for similarity search and cache‑key generation.<br>• **Community** – open‑source contributors and early adopters providing feedback & plugins. |
| **Key Activities** | • **Product development** – maintain compatibility with evolving LLM inference engines, add adaptive eviction policies, and improve hit‑rate ML models.<br>• **Performance testing** – benchmark latency & cost savings across model sizes (7B‑70B) and deployment patterns.<br>• **Security & privacy** – implement token‑level scoping, GDPR/CCPA compliance, and audit trails.<br>• **Documentation & SDKs** – generate client libraries for Python, Go, and JavaScript.<br>• **Customer success** – run PoCs, monitor cache metrics, and provide tuning recommendations. |
| **Key Partners** | • **vLLM** – primary inference engine partner; co‑development of cache hooks.<br>• **SGLang** – structured generation framework; joint performance benchmarks.<br>• **Redis Labs / ClickHouse** – backend storage partners for distributed cache persistence.<br>• **Cloud providers** (AWS, GCP, Azure) – preferred pricing for managed service deployment.<br>• **Axentx ecosystem** – integration with other Axentx products (e.g., RAG pipelines, prompt‑engineering tools). |
| **Unique Advantage** | Leverages Axentx’s massive request telemetry to predict cacheability, delivering **higher hit rates** than generic key‑value caches while staying **LLM‑aware** (token‑level granularity, prompt‑embedding similarity). |
